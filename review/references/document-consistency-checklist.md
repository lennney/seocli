# 文档间一致性检查清单

> 每轮评审后，用此清单检查所有文档间的不一致。高频问题标★。

## URL格式一致性 ★★★

所有文档中的URL必须统一。常见格式：
```
/games/{game}/          游戏页
/hints/{game}/today/    今日攻略（默认）
/hints/{game}/{date}/   指定日期攻略
/hints/{game}/how-to-play/  常青攻略
```

**检查方法：** 用search_files在所有文档中搜索URL模式，确认格式一致。

**常见不一致：**
- PRD用 `/{game}/hints/[date]`，IA用 `/hints/{game}/2026/06/04/`，用户故事用 `/daily/[game-slug]/today-hints-and-answers`
- 日期格式不一致：`2026/06/04` vs `2026-06-04`

## 游戏数量一致性 ★★

MVP游戏数量在所有文档中必须一致。砍掉游戏后：
- PRD功能需求
- 用户故事
- Sprint排期
- 原型（首页卡片、攻略预览、More Games区域）
- IA页面清单
- 首页统计面板数字

**检查方法：** 全局搜索被砍游戏的名字（如"WordHunt"），确认无残留引用。

## 优先级一致性 ★★

同一个功能在不同文档中的优先级必须一致：
- PRD标记P0的功能 → 用户故事也必须P0
- PRD标记P1的功能 → 不要在Sprint排期中排到MVP

**检查方法：** 对比PRD功能列表和用户故事优先级汇总表。

## 渲染策略一致性 ★★

SSG/SSR/ISR参数在PRD、IA、技术方案中必须一致：
- 首页：SSG+ISR(3600s)
- 游戏页：纯SSG
- 攻略页：按需revalidate（发布API触发）
- 归档页：纯SSG

**检查方法：** 全局搜索"ISR"、"SSG"、"SSR"、"revalidate"。

## 估时一致性 ★

同一功能在PRD和Sprint排期中的估时必须一致：
- PRD说"1-2天"，Sprint排了1周 → 要么改PRD要么改Sprint
- PRD说"3000+词"，Sprint说"2300+词" → 统一

## 变现触发条件一致性 ★

- PRD：月UV 1万→AdSense，月PV 5万→Mediavine
- Sprint排期：附录中不要写不同的标准
- 运营手册：与PRD一致

## 语言一致性

- PRD说"Phase 1只做英文" → 原型中不要出现中文
- 竞品分析不要引用与当前市场不符的论述（如做英文市场时引用"中文市场空白"）

## 技术栈一致性

- PRD/IA/Sprint排期的技术栈描述一致
- 运营手册不要引用已废弃的技术方案（如WordPress）

## 检查流程

```python
# 用search_files批量检查
1. 搜索被砍功能名 → 确认无残留
2. 搜索URL模式 → 确认格式统一
3. 搜索"ISR"/"SSG"/"SSR" → 确认渲染策略统一
4. 搜索"P0"/"P1" → 确认优先级对齐
5. 搜索数字（游戏数量/词库规模/错误次数） → 确认数字一致
6. 搜索技术栈关键词 → 确认无旧技术残留
```
