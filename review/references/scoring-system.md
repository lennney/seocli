# 评分体系详解

## 风险评分（1-5）

用于评估每个维度的风险程度。

### 定义

| 分数 | 等级 | 含义 | 后续动作 |
|------|------|------|---------|
| 5/5 | 🔴 极高风险 | 致命问题，不解决不应启动 | 立即停止，重新设计 |
| 4/5 | 🟠 高风险 | 严重问题，上线前必须解决 | P0优先级，本周解决 |
| 3/5 | 🟡 中等风险 | 需要注意，有workaround | P1优先级，两周内解决 |
| 2/5 | 🟢 低风险 | 小问题，不影响大局 | P2优先级，可延后 |
| 1/5 | ⚪ 无风险 | 没问题 | 无需行动 |

### 评分规则

1. **必须有理由**：每个分数必须附带核心质疑（一句话）
2. **必须有建议**：每个评分必须附带改进建议（一句话）
3. **一票否决**：某些维度出现5/5时，整体方案可能不可行
4. **趋势追踪**：多轮评审时，跟踪每个维度的分数变化

### 一票否决场景

| 场景 | 触发条件 | 后果 |
|------|---------|------|
| 内容农场 | RSS聚合+AI改写，无原创 | Google会惩罚，SEO归零 |
| 搜索量为0 | 目标关键词无搜索需求 | 假设不成立，需重新选词 |
| 技术不可行 | 需要的API/技术不存在 | 需重新设计技术方案 |
| 法律风险 | 违反版权/数据保护法规 | 可能被起诉 |
| 成本不可控 | 月成本>预算10倍 | 商业模式不成立 |

## 整体可行性评分

用于评估方案整体是否可行。

### 计算方式

```
整体可行性 = 10 - (所有维度平均风险分 - 1) * 2
```

或简单取：所有维度风险分的平均值，然后：
- 平均风险 1-2 → 可行性 4-5/5
- 平均风险 2-3 → 可行性 3-4/5
- 平均风险 3-4 → 可行性 2-3/5
- 平均风险 4-5 → 可行性 1-2/5

### 等级定义

| 分数 | 含义 | 后续动作 |
|------|------|---------|
| 4-5/5 | 成熟，可执行 | 直接进入开发 |
| 3-4/5 | 基本可行，需调整 | 解决P0问题后进入开发 |
| 2-3/5 | 问题较多，需重大修改 | 解决P0+P1问题后重新评审 |
| 1-2/5 | 不可行，需重新设计 | 进入Round 0方向探索 |

## SEO专属评分维度

### 关键词策略评分

| 分数 | 条件 |
|------|------|
| 1/5 | 搜索量>10k/月，竞争度低，意图匹配 |
| 2/5 | 搜索量5-10k/月，竞争度中等 |
| 3/5 | 搜索量1-5k/月，竞争度中等 |
| 4/5 | 搜索量<1k/月或竞争度高 |
| 5/5 | 搜索量<100/月或竞争度极高（DR90+头部） |

### 内容质量评分

| 分数 | 条件 |
|------|------|
| 1/5 | 原创内容，有真实体验，E-E-A-T完整 |
| 2/5 | AI辅助+人工审核，有专业背书 |
| 3/5 | AI改写+人工审核，但缺乏独特见解 |
| 4/5 | 纯AI改写，无人工审核 |
| 5/5 | 内容农场模式（RSS聚合+AI改写） |

### Google政策合规评分

| 分数 | 条件 |
|------|------|
| 1/5 | 完全合规，people-first内容 |
| 2/5 | 基本合规，AI辅助比例<30% |
| 3/5 | 边缘合规，AI内容比例30-60% |
| 4/5 | 高风险，AI内容比例>60% |
| 5/5 | 违规，规模化AI生成+无独特价值 |

## 评分校准

为避免评分不一致，建议：

1. **建立校准集**：收集5-10个典型案例，人工标注标准分数
2. **定期校准**：每10次评审后，回顾评分是否一致
3. **角色间校准**：不同角色对同一问题的评分差异不应>2分
4. **主持人裁决**：差异>2分时，主持人需裁决最终分数
