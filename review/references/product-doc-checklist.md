# 产品文档完整清单

按顺序产出，每个阶段的文档确认后再进入下一阶段。

## PM职责文档（开发团队直接使用）

以下是PM必须产出的核心文档，按顺序：

### 第一优先级（先出，用户确认后再做其他的）
1. **PRD** — 产品需求文档（功能需求/非功能需求/成功指标/里程碑/风险/决策记录）

### 第二优先级（PRD确认后）
2. **IA** — 信息架构（站点结构/页面层级/URL设计/渲染策略）
3. **HTML原型** — 可交互原型（Tailwind CDN，浏览器直接打开）

### 第三优先级（IA+原型确认后）
4. **用户故事** — User Stories + 验收标准（Given/When/Then格式）
5. **Sprint排期** — 项目排期（按Sprint组织，每周具体任务）

### 第四优先级（可选但推荐）
6. **竞品分析** — 竞品对比矩阵+差异化定位

### 不是PM的活（交给其他角色）
- 技术方案 → 技术负责人
- SEO策略 → SEO专家/运营
- 运营SOP → 运营总监
- UI设计稿 → 设计师

## 用户决策记录格式

PRD中需要有"待确认事项决策记录"表格：

```markdown
| # | 问题 | 决策 | 说明 |
|---|------|------|------|
| 1 | XXX | 决策内容 | 理由 |
```

每个重要决策需要附带对比分析（优势/劣势表格）。

## 文档间一致性检查清单

每次产出新文档或修改文档后，检查：

```
□ URL格式：PRD ↔ IA ↔ 用户故事 ↔ 原型
□ 功能数量：PRD ↔ 用户故事 ↔ Sprint排期 ↔ 原型
□ 优先级：PRD ↔ 用户故事
□ 数字（词库规模、时间估算等）：PRD ↔ Sprint排期
□ 技术参数（ISR间隔、性能指标等）：PRD ↔ IA ↔ 技术方案
□ 品牌名/产品名：所有文档+原型
□ 战略方向（市场定位、语言等）：PRD ↔ 竞品分析
□ localStorage/状态管理格式：Schema文档 ↔ 原型代码 ↔ PRD
□ 时间估算（开发天数等）：PRD ↔ Sprint排期
□ 渲染策略（SSG/SSR/ISR参数）：PRD ↔ IA
```
