# Claude Code 适配

## 并行机制

Claude Code 使用 Task 工具或多个子对话实现并行。

### 方式1: Task工具并行

```markdown
请同时执行以下3个评审任务：

Task 1: 你是产品经理。审核[方案]，给出风险评分(1-5)、核心质疑、建议。
Task 2: 你是运营总监。审核[方案]，给出风险评分(1-5)、核心质疑、建议。
Task 3: 你是SEO专家。审核[方案]，给出风险评分(1-5)、核心质疑、建议。
```

Claude Code会自动并行执行多个Task。

### 方式2: 顺序执行（如果并行受限）

```markdown
请依次扮演以下角色审核方案：
1. 产品经理 → 输出评分矩阵
2. 运营总监 → 输出评分矩阵
3. SEO专家 → 输出评分矩阵
然后汇总所有评分，形成矩阵。
```

### 方式3: 单prompt多角色

```markdown
请同时扮演PM、运营、SEO三个角色，从各自视角独立审核方案。
每个角色输出：风险评分(1-5) + 核心质疑 + 建议。
角色之间不要互相参考，确保独立性。
最后汇总为评分矩阵。
```

### 调研

```markdown
请搜索"[关键词]"的行业数据，返回：数据来源URL、关键数字、可信度标注。
```

### 注意事项

- Claude Code的Task工具支持并行，但可能受配置限制
- 单prompt多角色时，确保角色之间"独立"——可以在prompt中强调"角色之间不交流"
- Claude Code没有Hermes的delegate_task那样的隔离上下文，角色可能互相影响
- 建议在prompt中明确要求"每个角色的输出独立成段，不要混合"
