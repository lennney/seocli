# Codex 适配

## 并行机制

Codex（OpenAI）支持并行task执行。

### 方式1: 并行Task

```markdown
请同时执行以下评审任务：

[Task 1 - PM视角]
你是产品经理。从需求合理性角度审核以下方案...
输出：风险评分(1-5) + 核心质疑 + 建议

[Task 2 - 运营视角]
你是运营总监。从运营执行角度审核以下方案...
输出：风险评分(1-5) + 核心质疑 + 建议

[Task 3 - SEO视角]
你是SEO专家。从搜索优化角度审核以下方案...
输出：风险评分(1-5) + 核心质疑 + 建议
```

### 方式2: 单对话多角色

如果并行受限，可以在单个对话中依次触发：

```markdown
请依次审核方案：

Step 1: 扮演产品经理，输出评分
Step 2: 扮演运营总监，输出评分
Step 3: 扮演SEO专家，输出评分
Step 4: 汇总为评分矩阵
```

### 方式3: 使用Codex的subagent能力

如果Codex配置了subagent/spawn能力：

```markdown
请派出3个独立的评审agent：
1. PM Agent: 审核方案的产品可行性
2. 运营Agent: 审核方案的运营可行性
3. SEO Agent: 审核方案的SEO合规性

每个agent独立输出评分，最后汇总。
```

### 调研

```markdown
请搜索以下关键词的行业数据：
- "[关键词1]"
- "[关键词2]"

返回：来源URL、关键数字、可信度标注（✅已验证 / ⚠️可能对 / ❌未验证）
```

### 注意事项

- Codex的具体并行能力取决于版本和配置
- 如果无法真正并行，用"单对话多角色"方式，但要在prompt中强调角色独立性
- Codex的代码能力强，适合技术方案评审
- 产品/运营/SEO评审可能需要更多上下文引导
- 建议在prompt中提供角色定义和评分标准，减少模型自由发挥的空间

### Codex特有的优化

Codex擅长代码和技术，可以利用这一优势：

```markdown
技术评审专用prompt：
请作为技术负责人，审核以下架构方案：
1. 代码实现可行性（给出伪代码验证）
2. 性能瓶颈分析（给出量化估算）
3. 安全风险评估（列出OWASP Top 10相关项）
4. 技术债务评估（短期vs长期影响）

输出格式：
| 维度 | 风险评分(1-5) | 代码级证据 | 建议 |
```
